# 上下文压缩评估框架实现指南

## 概述

本文档详细介绍基于探针（probe-based）的上下文压缩质量评估框架，包括评估原理、探针设计、评分标准、代码实现和使用示例。

## 一、Probe-Based 评估的原理和优势

### 1.1 核心原理

Probe-based 评估通过向模型提出特定的探测性问题（probes），检验压缩后的上下文是否保留了关键信息。这种方法模拟了真实的对话延续场景，比简单的文本相似度指标更能反映压缩质量。

**评估流程：**
```
原始对话历史 → 压缩算法 → 压缩后上下文
                              ↓
                         探针问题 → 模型响应 → LLM Judge 评分
```

### 1.2 方法优势

1. **任务导向性**：直接测试压缩后上下文在实际任务中的可用性，而非抽象的文本相似度
2. **多维度评估**：从准确性、完整性、连续性等多个维度全面评估压缩质量
3. **可解释性**：每个评分都有明确的推理过程，便于诊断压缩算法的弱点
4. **真实场景模拟**：探针问题模拟用户在对话中可能提出的真实问题
5. **方法无关性**：可用于评估任何压缩算法，支持公平对比

### 1.3 与传统评估方法的对比

| 评估方法 | 优点 | 缺点 |
|---------|------|------|
| ROUGE/BLEU | 计算简单，速度快 | 无法评估语义保留，对改写敏感 |
| 语义相似度 | 考虑语义等价 | 无法评估任务相关性 |
| Probe-based | 任务导向，多维度，可解释 | 需要 LLM Judge，成本较高 |

## 二、四种探针类型详解

### 2.1 Recall Probes（回忆探针）

**设计目的**：测试模型对对话历史中具体事实细节的记忆能力。

**适用场景**：
- 检验技术细节（错误消息、版本号、命令）的保留
- 验证关键决策点的记录
- 测试时间线和因果关系的保持

**生成策略**：
```python
def generate_recall_probes(self) -> List[Probe]:
    probes = []

    # 提取原始错误信息
    if "original_error" in self.extracted_facts:
        probes.append(Probe(
            probe_type=ProbeType.RECALL,
            question="What was the original error message that started this debugging session?",
            ground_truth=self.extracted_facts["original_error"]
        ))

    # 提取版本决策
    version_pattern = r"version\s+(\d+\.\d+\.\d+)"
    match = re.search(version_pattern, self.history)
    if match:
        probes.append(Probe(
            probe_type=ProbeType.RECALL,
            question="What version of the dependency did we decide to use?",
            ground_truth=match.group(0)
        ))

    return probes
```

**示例问题**：
- "What was the exact error message we encountered?"
- "What Redis connection timeout value did we configure?"
- "Which authentication method did we initially try?"

**评分重点**：
- 事实准确性（文件路径、技术术语、数值）
- 细节完整性（是否遗漏关键信息）
- 无幻觉（不编造不存在的信息）

### 2.2 Artifact Probes（工件探针）

**设计目的**：测试模型对文件操作历史的追踪能力。

**适用场景**：
- 验证文件创建/修改/读取记录的完整性
- 检查代码变更细节的保留（函数名、变量名）
- 测试工作流程的连贯性

**生成策略**：
```python
def generate_artifact_probes(self) -> List[Probe]:
    probes = []

    if self.extracted_files:
        # 文件修改列表
        modified_files = [f for f in self.extracted_files if f["operation"] == "modified"]
        if modified_files:
            file_list = ", ".join([f["path"] for f in modified_files])
            probes.append(Probe(
                probe_type=ProbeType.ARTIFACT,
                question="Which files have we modified? Describe what changed in each.",
                ground_truth=file_list
            ))

        # 新创建的文件
        created_files = [f for f in self.extracted_files if f["operation"] == "created"]
        if created_files:
            probes.append(Probe(
                probe_type=ProbeType.ARTIFACT,
                question="What new files did we create in this session?",
                ground_truth=", ".join([f["path"] for f in created_files])
            ))

    return probes
```

**示例问题**：
- "Which files have we modified? Describe what changed in each."
- "What new configuration files did we create?"
- "Which test files did we examine but not change?"
- "What function names did we add to auth.py?"

**评分重点**：
- 文件列表的完整性
- 操作类型的准确性（创建/修改/读取）
- 变更描述的详细程度
- 关键标识符（函数名、变量名）的记忆

### 2.3 Continuation Probes（延续探针）

**设计目的**：测试模型能否基于压缩上下文无缝继续工作，而不需要重新获取信息。

**适用场景**：
- 验证工作状态的保留
- 检查待办任务的追踪
- 测试下一步行动的合理性

**生成策略**：
```python
def generate_continuation_probes(self) -> List[Probe]:
    probes = []

    # 基于提取的下一步信息
    if "next_steps" in self.extracted_facts:
        probes.append(Probe(
            probe_type=ProbeType.CONTINUATION,
            question="What should we do next?",
            ground_truth=self.extracted_facts["next_steps"]
        ))

    # 基于未完成的任务
    probes.append(Probe(
        probe_type=ProbeType.CONTINUATION,
        question="What was left incomplete from our last step?",
        context_reference="Check if agent can identify incomplete work"
    ))

    return probes
```

**示例问题**：
- "What should we do next?"
- "What tests are still failing and why?"
- "What was left incomplete from our last step?"
- "Can you continue implementing the authentication feature?"

**评分重点**：
- 能否在不重新读取文件的情况下继续工作
- 对待办任务的感知
- 对当前工作状态的理解
- 下一步建议的合理性和可操作性

### 2.4 Decision Probes（决策探针）

**设计目的**：测试模型对决策推理链和决策依据的保留能力。

**适用场景**：
- 验证技术选型理由的记忆
- 检查权衡分析的保留
- 测试替代方案的记录

**生成策略**：
```python
def generate_decision_probes(self) -> List[Probe]:
    probes = []

    for decision in self.extracted_decisions[:3]:  # 取前3个决策
        probes.append(Probe(
            probe_type=ProbeType.DECISION,
            question=f"Why did we decide to {decision['decision']}? What alternatives did we consider?",
            ground_truth=decision.get("reasoning", ""),
            context_reference=decision["context"]
        ))

    return probes
```

**示例问题**：
- "Why did we choose connection pooling over per-request connections?"
- "What alternatives did we consider for the Redis timeout issue?"
- "We discussed several options for authentication. What did we decide and why?"

**评分重点**：
- 决策结果的准确性
- 决策理由的完整性
- 替代方案的记录
- 权衡因素的保留

## 三、六个评估维度的评分标准

### 3.1 Accuracy（准确性）

**维度定义**：响应中事实、技术细节和代码引用的正确性。

**子标准**：

| 标准 ID | 评估问题 | 权重 | 0分 | 3分 | 5分 |
|---------|---------|------|-----|-----|-----|
| accuracy_factual | 事实、文件路径和技术细节是否正确？ | 0.6 | 完全错误或编造 | 大部分准确，有小错误 | 完全准确 |
| accuracy_technical | 代码引用和技术概念是否正确？ | 0.4 | 重大技术错误 | 基本正确，有小问题 | 技术上精确 |

**评分示例**：
```
问题：What was the Redis connection timeout we configured?
响应：We set it to 5000ms in redis.conf

Ground Truth：timeout = 5000 in redis.conf

评分：
- accuracy_factual: 5.0 (数值和文件名完全正确)
- accuracy_technical: 5.0 (技术术语使用正确)
```

### 3.2 Context Awareness（上下文感知）

**维度定义**：响应是否反映了对话状态和工件访问历史。

**子标准**：

| 标准 ID | 评估问题 | 权重 | 0分 | 3分 | 5分 |
|---------|---------|------|-----|-----|-----|
| context_conversation_state | 响应是否反映当前对话状态？ | 0.5 | 无先前上下文感知 | 一般感知，有缺口 | 完全感知对话历史 |
| context_artifact_state | 响应是否反映访问过的文件/工件？ | 0.5 | 无工件感知 | 部分工件感知 | 完整工件状态感知 |

**评分示例**：
```
问题：What should we do next?
响应：We should continue testing the authentication fix we just implemented in auth.py

评分：
- context_conversation_state: 5.0 (知道刚完成了认证修复)
- context_artifact_state: 5.0 (记得修改了 auth.py)
```

### 3.3 Artifact Trail（工件追踪）

**维度定义**：对文件操作历史和代码细节的记忆能力。

**子标准**：

| 标准 ID | 评估问题 | 权重 | 0分 | 3分 | 5分 |
|---------|---------|------|-----|-----|-----|
| artifact_files_created | 是否知道创建了哪些文件？ | 0.3 | 无知识 | 知道大部分文件 | 完美知识 |
| artifact_files_modified | 是否知道修改了哪些文件及变更内容？ | 0.4 | 无知识 | 知道大部分修改 | 完美知识 |
| artifact_key_details | 是否记得函数名、变量名、错误消息？ | 0.3 | 无记忆 | 记得大部分关键细节 | 完美记忆 |

**注意**：根据评估数据，这是所有压缩方法的普遍弱项（平均分 2.19-2.45），建议使用独立的工件追踪系统。

### 3.4 Completeness（完整性）

**维度定义**：响应是否全面回答了问题的所有部分。

**子标准**：

| 标准 ID | 评估问题 | 权重 | 0分 | 3分 | 5分 |
|---------|---------|------|-----|-----|-----|
| completeness_coverage | 响应是否涵盖问题的所有部分？ | 0.6 | 忽略大部分内容 | 涵盖大部分内容 | 全面涵盖所有内容 |
| completeness_depth | 是否提供了足够的细节？ | 0.4 | 肤浅或缺失细节 | 足够的细节 | 全面的细节 |

### 3.5 Continuity（连续性）

**维度定义**：能否在不重新获取信息的情况下继续工作。

**子标准**：

| 标准 ID | 评估问题 | 权重 | 0分 | 3分 | 5分 |
|---------|---------|------|-----|-----|-----|
| continuity_work_state | 能否在不重新获取信息的情况下继续？ | 0.4 | 无法继续，需重新获取所有上下文 | 可继续，需少量重新获取 | 可无缝继续 |
| continuity_todo_state | 是否保持对待办任务的感知？ | 0.3 | 丢失所有 TODO | 良好感知，有些缺口 | 完美任务感知 |
| continuity_reasoning | 是否保留先前决策的理由？ | 0.3 | 无推理记忆 | 一般记得推理 | 优秀保留 |

### 3.6 Instruction Following（指令遵循）

**维度定义**：响应是否遵循了格式和约束要求。

**子标准**：

| 标准 ID | 评估问题 | 权重 | 0分 | 3分 | 5分 |
|---------|---------|------|-----|-----|-----|
| instruction_format | 响应是否遵循请求的格式？ | 0.5 | 忽略格式 | 基本遵循格式 | 完美遵循格式 |
| instruction_constraints | 响应是否尊重声明的约束？ | 0.5 | 违反约束 | 基本尊重约束 | 完全尊重约束 |

### 3.7 维度得分计算

每个维度的得分是其子标准的加权平均：

```python
dimension_score = Σ(criterion_score × criterion_weight) / Σ(criterion_weight)
```

总体得分是所有维度得分的简单平均（无权重）：

```python
aggregate_score = Σ(dimension_score) / num_dimensions
```

### 4.2 核心类详解

#### 4.2.1 ProbeGenerator（探针生成器）

**职责**：从对话历史中提取信息并生成探针问题。

**关键方法**：

```python
class ProbeGenerator:
    def __init__(self, conversation_history: str):
        self.history = conversation_history
        self.extracted_facts = self._extract_facts()      # 提取事实
        self.extracted_files = self._extract_files()      # 提取文件操作
        self.extracted_decisions = self._extract_decisions()  # 提取决策

    def generate_probes(self) -> List[Probe]:
        """生成所有类型的探针"""
        probes = []
        probes.extend(self._generate_recall_probes())
        probes.extend(self._generate_artifact_probes())
        probes.extend(self._generate_continuation_probes())
        probes.extend(self._generate_decision_probes())
        return probes
```

**信息提取策略**：

1. **事实提取** (`_extract_facts`)：
   - 使用正则表达式匹配错误消息
   - 提取 TODO 和下一步计划
   - 识别关键技术决策

2. **文件提取** (`_extract_files`)：
   - 匹配文件操作动词（modified, created, read）
   - 提取文件路径和扩展名
   - 去重并分类操作类型

3. **决策提取** (`_extract_decisions`)：
   - 识别决策关键词（decided to, chose, will use）
   - 提取决策内容和上下文
   - 限制数量避免过多探针

**生产环境改进建议**：
- 使用 NER（命名实体识别）替代正则表达式
- 集成代码解析器提取函数名和变量名
- 使用 LLM 进行更智能的信息提取

#### 4.2.2 CompressionEvaluator（评估器）

**职责**：使用 LLM Judge 评估探针响应质量。

**核心流程**：

```python
class CompressionEvaluator:
    def evaluate(self, probe: Probe, response: str, compressed_context: str) -> EvaluationResult:
        # 1. 获取相关评分标准
        criteria = self._get_criteria_for_probe(probe.probe_type)

        # 2. 逐个标准评估
        criterion_results = []
        for criterion in criteria:
            result = self._evaluate_criterion(criterion, probe, response, compressed_context)
            criterion_results.append(result)

        # 3. 计算维度得分
        dimension_scores = self._calculate_dimension_scores(criterion_results)

        # 4. 计算总分
        aggregate_score = sum(dimension_scores.values()) / len(dimension_scores)

        return EvaluationResult(...)
```

**标准选择逻辑** (`_get_criteria_for_probe`)：

```python
def _get_criteria_for_probe(self, probe_type: ProbeType) -> List[Dict]:
    criteria = []

    # 所有探针都评估准确性和完整性
    criteria.extend(RUBRIC_CRITERIA["accuracy"])
    criteria.extend(RUBRIC_CRITERIA["completeness"])

    # 根据探针类型添加特定标准
    if probe_type == ProbeType.ARTIFACT:
        criteria.extend(RUBRIC_CRITERIA["artifact_trail"])
    elif probe_type == ProbeType.CONTINUATION:
        criteria.extend(RUBRIC_CRITERIA["continuity"])
    elif probe_type == ProbeType.RECALL:
        criteria.extend(RUBRIC_CRITERIA["context_awareness"])
    elif probe_type == ProbeType.DECISION:
        criteria.extend(RUBRIC_CRITERIA["context_awareness"])

    # 所有探针都评估指令遵循
    criteria.extend(RUBRIC_CRITERIA["instruction_following"])

    return criteria
```

**更新流程**：

```python
def update_from_span(self, new_content: str) -> str:
    # 1. 从新内容中提取信息
    new_info = self._extract_from_content(new_content)

    # 2. 与现有部分合并
    self._merge_sections(new_info)

    # 3. 生成格式化摘要
    return self._format_summary()
```

**合并策略** (`_merge_sections`)：
- **Intent**：仅在为空时更新
- **Files Modified/Read**：按路径去重后追加
- **Decisions**：直接追加
- **Current State**：最新的覆盖旧的
- **Next Steps**：追加，格式化时取最后5个

def estimate_tokens(text: str) -> int:
    """
    简化的 token 估算
    生产环境应使用模型特定的 tokenizer

    enc = tiktoken.encoding_for_model("gpt-4")

#### evaluate_compression()

```python
def evaluate_compression(
    original_history: str,
    compressed_context: str,
    model_response_fn: callable
) -> Dict:
    """
    完整的压缩评估流程

    Args:
        original_history: 原始对话历史
        compressed_context: 压缩后的上下文
        model_response_fn: 函数(context, question) -> response

    Returns:
        包含得分和建议的摘要
    """
    # 1. 生成探针
    generator = ProbeGenerator(original_history)
    probes = generator.generate_probes()

    # 2. 评估每个探针
    evaluator = CompressionEvaluator()
    for probe in probes:
        response = model_response_fn(compressed_context, probe.question)
        evaluator.evaluate(probe, response, compressed_context)

    # 3. 生成摘要和建议
    summary = evaluator.get_summary()
    summary["recommendations"] = _generate_recommendations(summary)

    return summary
```

### 5.5 集成到生产系统

```python
import openai
from compression_evaluator import evaluate_compression, estimate_tokens

class ProductionCompressionEvaluator:
    def __init__(self, api_key: str, judge_model: str = "gpt-4"):
        self.api_key = api_key
        self.judge_model = judge_model
        openai.api_key = api_key

    def model_response_fn(self, context: str, question: str) -> str:
        """实际的 LLM 调用"""
        response = openai.ChatCompletion.create(
            model="gpt-4",
            messages=[
                {"role": "system", "content": f"Context:\n{context}"},
                {"role": "user", "content": question}
            ],
            temperature=0.0
        )
        return response.choices[0].message.content

    def evaluate_with_metrics(self, original: str, compressed: str) -> dict:
        """评估并返回详细指标"""
        # 基础指标
        original_tokens = estimate_tokens(original)
        compressed_tokens = estimate_tokens(compressed)
        compression_ratio = compressed_tokens / original_tokens

        # 质量评估
        quality_summary = evaluate_compression(
            original_history=original,
            compressed_context=compressed,
            model_response_fn=self.model_response_fn
        )

        # 综合报告
        return {
            "compression_metrics": {
                "original_tokens": original_tokens,
                "compressed_tokens": compressed_tokens,
                "compression_ratio": compression_ratio,
                "tokens_saved": original_tokens - compressed_tokens
            },
            "quality_metrics": quality_summary,
            "efficiency_score": self._calculate_efficiency(
                quality_summary['average_score'],
                compression_ratio
            )
        }

    def _calculate_efficiency(self, quality_score: float, ratio: float) -> float:
        """计算效率得分：质量 vs 压缩率的平衡"""
        # 归一化压缩率（假设目标是 0.3）
        target_ratio = 0.3
        ratio_score = 1.0 - abs(ratio - target_ratio) / target_ratio
        ratio_score = max(0, min(1, ratio_score)) * 5  # 转换为 0-5 分

        # 综合得分：质量占 70%，压缩率占 30%
        return quality_score * 0.7 + ratio_score * 0.3

# 使用示例
evaluator = ProductionCompressionEvaluator(api_key="your-api-key")
report = evaluator.evaluate_with_metrics(original_history, compressed_context)

print(f"Compression Ratio: {report['compression_metrics']['compression_ratio']:.2%}")
print(f"Tokens Saved: {report['compression_metrics']['tokens_saved']}")
print(f"Quality Score: {report['quality_metrics']['average_score']:.2f}")
print(f"Efficiency Score: {report['efficiency_score']:.2f}")

class ArtifactTracker:
    def __init__(self):
        self.files_created = []
        self.files_modified = {}
        self.files_read = []

# 使用缓存避免重复评估
import hashlib
import json

class CachedEvaluator(CompressionEvaluator):
    def __init__(self, cache_file="eval_cache.json"):
        super().__init__()
        self.cache_file = cache_file
        self.cache = self._load_cache()

    def _cache_key(self, probe, response):
        content = f"{probe.question}|{response}"
        return hashlib.md5(content.encode()).hexdigest()

    def evaluate(self, probe, response, compressed_context):
        key = self._cache_key(probe, response)
        if key in self.cache:
            return self.cache[key]

        result = super().evaluate(probe, response, compressed_context)
        self.cache[key] = result
        self._save_cache()
        return result
```

```python
import random

def sampled_evaluation(original_history, compressed_context,
                       model_fn, sample_rate=0.3):
    """只评估部分探针以降低成本"""
    generator = ProbeGenerator(original_history)
    all_probes = generator.generate_probes()

    # 确保每种类型至少有一个探针
    sampled = []
    for probe_type in ProbeType:
        type_probes = [p for p in all_probes if p.probe_type == probe_type]
        if type_probes:
            sampled.append(random.choice(type_probes))

    # 随机采样其余探针
    remaining = [p for p in all_probes if p not in sampled]
    sample_size = int(len(remaining) * sample_rate)
    sampled.extend(random.sample(remaining, sample_size))

    # 评估
    evaluator = CompressionEvaluator()
    for probe in sampled:
        response = model_fn(compressed_context, probe.question)
        evaluator.evaluate(probe, response, compressed_context)

    return evaluator.get_summary()
```
